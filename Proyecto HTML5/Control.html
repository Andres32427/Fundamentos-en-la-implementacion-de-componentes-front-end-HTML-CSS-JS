Etiquetas Control
Etiquetas Descripcion Propiedades Ejemplo
< button> En esta etiqueta se trata de la creacion de un boton interactivo en base al desarrollo de una pagina web por ende dicho boton interactivo podria llegar a tener algun tipo de informacion, texto, palabra,etc. disable:atributo booleano que desactiva el boton haciendolo nada interactivo
form:se logra asociar el boton con un formulario requerido
< command> Esta etiqueta se utiliza para definir un botón command que un usuario puede invocar ya sea un boton, una casilla de verificacion o elemento del menu. type: Especifica el tipo de comando.
label: Define el texto del comando.
icon: URL de un icono para el comando.
disabled: Desactiva el comando.
checked: indica si esta marcado (para checkbox o radio).
< datalist> Se utiliza para especificar en un input una lista pre definida de opciones. id: Asocia la lista con un campo de entrada.
option: Define cada una de las opciones dentro de la lista.
< dialog> Se utiliza para definir una caja o ventana de dialogo. open: Muestra el cuadro de dialogo.
returnvalue: Establece o recupera el valor de retorno al cerrar el dialogo.

Este es un cuadro de diálogo.
< fieldset> Se utiliza para agrupar controles relacionados dentro de un formulario HTML, proporcionando una estructura lógica y visual. disable: Desactiva todos los controles dentro del grupo.
form: Asocia el < fieldset> como un formulario especifico.
name: Nombre del < fieldset>.

< form> Se utiliza para crear un formulario en HTML que recoge datos ingresados por el usuario. action: URL a la que se envian los datos del formulario.
method: Metodo HTTP (GET, POST) para enviar datos.
enctype: Tipo de codificacion de los datos.
autocomplete: Si se debe autocompletar los campos (on, off).
novalidate: Desactiva la validacion del formulario.

< input> Se utiliza para crear diferentes tipos de campos de entrada en un formulario web. Es un elemento vacío, por lo que no tiene una etiqueta de cierre. type: Especifica el tipo de campo de entrada.
name: Define el nombre del campo.
value: Establece el valor inicial.
placeholder: Texto de ayuda que aparece en el campo.
required: Campo obligatorio.
disabled: Desactiva el campo.
readonly: Solo lectura.
< keygen> Se utiliza para generar un par de claves criptográficas (pública y privada) y permite al usuario enviar la clave pública junto con un formulario. name: Asocia la clave publica con un nombre en el formulario.
challenge: Incluye un desafio en la generacion de claves.
keytype: Define el tipo de algoritmo criptografico (como RSA).
autofocus: Enfoca el campo automaticamente.
disabled: Desactiva el campo.
form: Asocia el campo con un formulario especifico.
< label> Se utiliza para definir etiquetas asociadas a elementos de formulario. for: Asocia la etiqueta con el elemento de formulario especificado por su id.
form: Asocia la etiqueta con un formulario especifico.
contenido: Puede contener texto o elementos en linea.
< legend> Es utilizada dentro de un formulario, en combinación con < fieldset>, para proporcionar un título o descripción para un grupo de controles de formulario, lo que ayuda a organizar y describir conjuntos de controles relacionados. contenido: Texto o elemento en linea.
Atributos globales: Soporta atributos como class, id, style.
Información personal
< meter> Se utiliza para representar un valor escalar dentro de un rango conocido. Es útil para mostrar mediciones como la capacidad de una batería, el uso de almacenamiento, la puntuación de un examen, entre otros. min: Define el valor minimo del rango (por defecto 0).
max: Define el valor maximo del rango (por defecto 1).
value: El valor actual a mostar en el medidor.
low: Define el valor considerado bajo.
high: Define el valor considerado alto.
optimum: El valor optimo dentro del rango. 		2 out of 10

< optgroup> Se utiliza para agrupar opciones relacionadas dentro de un menú desplegable < select>. Esto permite organizar mejor las opciones cuando hay muchas, ayudando al usuario a navegar y seleccionar una opción. label: Nombre visible del grupo de opciones.
disabled: Desactiva todas las opciones dentro del grupo.


< option> Se utiliza dentro de un elemento < select> o < datalist> para definir una opción que el usuario puede seleccionar. Es parte integral de la creación de listas desplegables en formularios. value: Define el valor que se enviará con el formulario si se selecciona esta opción.
selected: Indica que esta opción es la seleccionada por defecto.
disabled: Desactiva la opción, haciendo que no se pueda seleccionar.
label: Define un texto corto alternativo para la opción.
< output> Se utiliza para mostrar los resultados de un cálculo o la respuesta a una acción del usuario en un formulario. for: Especifica los elementos asociados cuyo valor afecta al resultado.
form: Asocia el elemento < output> con un formulario especifico.
name: Define un nombre para el valor del resultado.

< progress> Se utiliza para mostrar el progreso de una tarea que está en curso. value: Define el valor actual del progreso.
max: Define el valor maximo de progreso.
label: Atributo aria que proporciona una descripcion accesible. 		32%
<;script>; Se utiliza para incluir y/o ejecutar código JavaScript en un documento HTML. Puede ser utilizada para agregar scripts internos dentro de la propia página o para enlazar scripts externos. src: URL del archivo de script externo.
async: Ejecuta el script de forma asincrona.
defer: Ejecuta el script después de que la página se haya analizado.
type: Tipo de script.
crossorigin: Gestión de solicitudes de recursos desde otros orígenes.
integrity: Verificacion de integridad del script.

Bienvenido a la parte de la estructura de control
< select> Se utiliza para crea un menú desplegable en un formulario, permitiendo que el usuario seleccione una opción de una lista de opciones. name: Especifica el nombre del control, que se envía junto con los datos del formulario.
id: Asigna un identificador único al elemento, útil para asociarlo con etiquetas < label> o para manipulación con JavaScript y CSS.
multiple: Permite seleccionar múltiples opciones. Si está presente, el usuario puede elegir más de una opción en la lista.
size: Define el número de opciones visibles en la lista sin necesidad de desplazarse.
required: Indica que el menú debe tener una opción seleccionada antes de enviar el formulario.
< textarea> Se utiliza para crear un área de entrada de texto que permite a los usuarios escribir múltiples líneas de texto. cols: Define el número de columnas (ancho) del área de texto.
rows: Define el número de filas (altura) del área de texto.
placeholder: Muestra un texto de sugerencia dentro del área de texto cuando está vacío.
maxlength: Establece el número máximo de caracteres que se pueden ingresar.
readonly: Hace que el área de texto sea de solo lectura, impidiendo su modificación.
disabled: Desactiva el área de texto, impidiendo que se interactúe con ella.
required: Indica que el campo es obligatorio y debe ser llenado antes de enviar el formulario.
wrap: Especifica cómo el texto debe ajustarse cuando se envía al servidor.
class: Asigna una o más clases CSS para aplicar estilos específicos al área de texto.
id: Asigna un identificador único al elemento para su selección y manipulación con CSS o JavaScript.
style: Permite definir estilos en línea para personalizar la apariencia del área de texto.
